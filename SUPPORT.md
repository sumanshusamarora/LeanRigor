# Support

LeanRigor is an early-stage open-source project maintained on a best-effort basis.

## Where to ask

- Use a **bug report** for reproducible product defects.
- Use a **feature request** for product proposals and use cases.
- Use a **documentation issue** for unclear, missing, or inaccurate documentation.
- Use a **provider or adapter request** for Claude Code, OpenCode, Codex, or other execution integrations.
- Use a **workspace or integration issue** for worktree, lease, revision, integration, or user-working-tree safety problems.

When GitHub Discussions is enabled, general questions and design conversations should go there rather than into bug issues.

## What to include

Please provide the LeanRigor version or commit, installation type, Node.js version, operating system, coding-agent version, reproduction steps, expected behaviour, actual behaviour, and sanitised logs where relevant.

Never post credentials, tokens, private source code, customer data, or provider transcripts containing sensitive information.

## Response expectations

There is no guaranteed response time or service-level agreement. The maintainer will prioritise:

1. security and user-working-tree safety;
2. installation and release blockers;
3. reproducible workflow correctness defects;
4. documentation and contributor blockers;
5. feature proposals aligned with LeanRigor's product principles.

## Private contact

For security or conduct reports, use GitHub private vulnerability reporting when available or contact the repository owner privately through their GitHub profile. Public issues should not be used for sensitive reports.
