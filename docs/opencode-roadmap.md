# OpenCode Adapter Roadmap

The OpenCode adapter will reuse the same capability profiles, execution graph, file ownership model, workflow state, and safety policy.

Planned work:

- Resolve profiles to provider/model identifiers.
- Use the configured OpenCode small model for triage.
- Map lifecycle events to LeanRigor hooks.
- Install commands and specialist agents.
- Validate model availability where supported.
- Preserve user freedom to use Anthropic, OpenAI, Gemini, local, or other providers.

No OpenCode-specific model names are embedded in the portable core.
